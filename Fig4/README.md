# Figure 4 — Diversity of evolutionary histories and tissue tropisms among zebrafish viruses

Code to reproduce the curated fish-associated virus dataset, the percent-identity and
heat-tree panels of **Figure 4**, the related supplemental panels, and some related data products used
in the RNAquarium portal.

The pipeline starts from per-contig virus taxonomy + a Salmon expression matrix (produced by
upstream assembly/quantification, not included here — see **Data availability**) and a small
set of hand-curated tables (included). It runs in **R**.

 - The following readme and accompanying code and DATA_MANIFEST are for generating panels A, B & D
 - Panel C phylogenetic tree images made using Geneious with inputs saved (`HA_forfigureAA_` & `PB2_forfigureAA_` files)
 - Panel E schematic made using `scRNAseq_DBPV_analysis.R` code example

## Pipeline (run in order)

| step | file | what it does |
|------|------|--------------|
| 1 | `01_curate.ipynb` | From the metatranscriptome virus taxonomy + the curation tables, builds the canonical curated contig set, per-virus table, a fasta, and a versioning contig crosswalk. |
| 2a | `02a_reduce_matrix.R` | Heavy step (run as a batch/`Rscript` job): reduces the full Salmon count matrix to the curated contigs. |
| 2b | `02b_counts.ipynb` | Collapses contig counts to per-virus counts and joins run-level metadata → the supplemental counts table. |
| 3 | `03_figures.ipynb` | Figure 4 metacoder heat-tree (Fig 4A), the percent-identity dot plots (Fig 4B + supplemental), Figure 4D (7 viruses by tissue), and the supplemental per-metadata bar panels. |
| 4 | `04_portal.ipynb` | Builds the web-portal artifacts from the same curated set: the interactive radial heat-tree JSON and the portal curated TSV + fasta. |

Dependency order: **1 → 2a → 2b → 3**; **4** depends only on **1**.

## Setup

1. **Set the data directory.** Each notebook/script has a `CONFIG` block at the top with a
   `<DATA_DIR>` placeholder. Set it to the folder containing the large input files (below).
2. **R packages:** `tidyverse`, `data.table`, `metacoder`, `taxize`, `jsonlite`, `lubridate`,
   and optionally `pals` (for two of the supplemental panel palettes). `02a` benefits from a
   large-memory session.
3. `03`'s heat-trees query NCBI taxonomy via `taxize`; set an `ENTREZ_KEY` environment variable
   to raise the rate limit. (The call is wrapped in a retry loop.)

## Included curated inputs (the human-curated judgment behind the dataset)

- `cluster_curation.tsv` — accepted virus clusters → curated name → Category. The set of
  clusters here defines what counts as fish-associated.
- `rejected_clusters.tsv` — clusters considered and excluded as non-fish. *(Regenerated on the
  first run of `01`; add it here for the record.)*
- `rejected_taxnames.tsv` — off-target taxa removed from within accepted clusters.
- `contig_split_overrides.tsv` — specific contigs reassigned to a second curated virus.
- `rescue_sequences.tsv` — sequences for 4 contigs dropped upstream as adapters, preserved here.
- `fig4b_family_curation.tsv` — Category | Family | Virus labels for the Figure 4B table.

## Inputs NOT included (large / upstream) — Data availability

Point `<DATA_DIR>` at a copy of these:

- two per-contig virus taxonomy tables (generated by the metatranscriptme pipeline, also 75k examples zipped in this folder)
- the full Salmon expression matrix (found in RNAquarium portal under *Data - "Virus Abundance Matrix"*),
- the run-level sample metadata table (found in RNAquarium portal under *Data - "SRA Metadata"*).

## Outputs

- Curated virus contig table + per-virus table + fasta (also the portal dataset).
- Supplemental counts-per-run + metadata table.
- Figure 4 panels: heat-tree(s), percent-identity dot plots (4B + supplemental), 4D tissue
  panel, and the supplemental tissue/devstage/year/country bar panels.
- Portal artifacts: interactive radial heat-tree JSON + the dated curated TSV/fasta (from `04`).
